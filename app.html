みまもるくん（アプリの説明）

GitHub のコミット数やブランチ作成回数を取得して記録するデスクトップアプリです。

概要（何をするアプリか）

みまもるくんは、あなたの GitHub アカウントで認証した後、選択したリポジトリの コミット数やブランチ情報を取得して、ローカルに JSON ファイルとして保存します。 チームの貢献度や活動記録の可視化に便利です。

主な機能

  • GitHub での OAuth 認証（Device Flow を利用した簡単ログイン）
  • リポジトリを登録・管理（最大5件）
  • 選択したリポジトリからコミット数・ブランチ数などを取得し JSON として保存
  • 取得データは `github-data.json` というファイル名で保存されます

セットアップ（概要）

開発元の手順に従い、Node.js と GitHub OAuth アプリの設定が必要です。簡単な手順：

  1. リポジトリをクローンして `npm install` を実行
  2. GitHub 側で OAuth App を作成し、Client ID / Client Secret を取得
  3. ルートに `.env` を作成して `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET` を設定
  4. `npm run dev` で開発モード起動

使い方（簡単な流れ）

  1. アプリを起動して「GitHubでログイン」を押す
  2. 表示されたコードをブラウザの GitHub 認証画面に入力して承認
  3. 認証後、リポジトリ管理画面で取得対象のリポジトリを選択
  4. データを取得して保存ボタンを押すと `github-data.json` が生成される

出力されるデータ形式（例）

{
  "owner/repo-name": {
    "commits": {
      "total": 120,
      "byUser": { "user1": 80, "user2": 40 }
    },
    "branches": { "total": 10, "byUser": { "user1": 6, "user2": 4 } },
    "fetched_at": "2026-05-29T07:00:00.000Z"
  }
}

注意事項

  • `.env` に含まれる `GITHUB_CLIENT_SECRET` は絶対に公開リポジトリへプッシュしないでください。
  • `github-data.json` にはリポジトリ情報が含まれるため取扱に注意してください。
  • 本ページは説明目的です。本体のビルドや配布は公式リポジトリの手順に従ってください。

さらに詳しい実装や開発手順は元のリポジトリの README を参照してください。 （`seisaku2026/mimamorukun/README.md`）
